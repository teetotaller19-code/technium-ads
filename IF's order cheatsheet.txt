Табличка-«шпаргалка» для порядка if-ов
Сверху вниз обычно так:
1. Невозможные состояния (None, пусто, нет ключа, не тот тип)
2. Инварианты безопасности (чтобы дальше не упасть: деление на 0, пустой стек, выход за границы)
3. Валидация входа (формат, допустимые символы)
4. Бизнес-логика / основная логика
5. Финальная проверка результата (например, “стек пуст?”)
